# 用例与风格预设

这份文件用于设计新的学习系统时快速选型。每个用例都可以复用同一套引擎，只需要替换技能图谱、题库、rubric 和 teacher 风格。

## 用例设计

### 哲学原典研读

适合对象：柏拉图、康德、胡塞尔、海德格尔、维特根斯坦等原典。

技能图谱：

- 按章节、论证步骤、核心概念拆分
- 每个技能点绑定具体段落或术语
- 前置关系体现“概念依赖”和“论证依赖”

题库重点：

- 概念辨析
- 误读纠正
- 论证重构
- 反例和边界判断
- 跨章节概念关联

推荐风格：苏格拉底式、研究生研讨式、严格考官式。

### 经济学学习

适合对象：微观经济学、宏观经济学、政治经济学、行为经济学。

技能图谱：

- 按模型、假设、机制、图形、政策含义拆分
- 每个技能点明确“前提 -> 推理 -> 结论”
- 重要模型之间建立桥接技能点

题库重点：

- 模型假设识别
- 机制解释
- 图表解读
- 政策情境应用
- 反事实分析

推荐风格：模型教练式、政策简报式、案例分析式。

### 交易与市场训练

适合对象：交易系统、风险管理、宏观交易、量化策略、复盘训练。

技能图谱：

- 按市场结构、信号、风险、执行、复盘拆分
- 技能点必须绑定可观察行为或判断规则
- 前置关系优先体现“先风险，后策略”

题库重点：

- 场景判断
- 风险边界
- 交易计划审查
- 事后复盘
- 常见心理偏差识别

推荐风格：交易教练式、风控官式、复盘审计式。

注意：这类系统只应训练判断和复盘流程，不应输出具体投资建议或保证收益。

### 政治学与公共议题

适合对象：政治理论、国际关系、比较政治、公共政策。

技能图谱：

- 按概念、制度、行为者、机制、案例拆分
- 区分规范性问题和经验性问题
- 把理论框架与案例材料建立桥接

题库重点：

- 概念定义
- 制度比较
- 因果机制
- 立场论证
- 案例迁移

推荐风格：辩论庭式、政策备忘录式、比较案例式。

### 历史学习

适合对象：通史、专题史、思想史、经济史、战争史。

技能图谱：

- 按时间线、事件链、结构因素、人物、史料问题拆分
- 前置关系体现因果链和背景知识
- 技能点不要只等于年份记忆，要包含解释任务

题库重点：

- 时间线重构
- 因果解释
- 史料辨析
- 多因素比较
- 反事实边界

推荐风格：历史叙事式、档案分析式、因果审问式。

### 数学学习

适合对象：高等数学、线性代数、概率论、实分析、抽象代数。

技能图谱：

- 按定义、定理、证明方法、典型题型拆分
- 前置关系必须严格
- 每个技能点绑定可检验的证明或计算能力

题库重点：

- 定义复述
- 反例构造
- 证明补全
- 计算步骤
- 定理适用条件

推荐风格：严格证明式、错题诊断式、定理地图式。

### 写作训练

适合对象：论文写作、非虚构写作、商业写作、小说写作、英语写作。

技能图谱：

- 按选题、结构、段落、论证、风格、修改拆分
- 每个技能点绑定可产出的文本动作
- 练习结果应保存样稿和修改理由

题库重点：

- 结构诊断
- 段落改写
- 论证补强
- 语气转换
- 读者视角检查

推荐风格：写作工作坊式、编辑审稿式、产品文案式。

## 风格预设

### 苏格拉底式

适合：哲学、政治理论、概念密集型学科。

特点：

- 以问题推进，不急于给结论
- 每次追问一个概念差异
- 强调学习者自己说出定义和边界
- 适合概念课和示范练习

teacher 调整：

- `main_teacher.md` 强化追问
- `assistant_teacher.md` 负责给类比和缓冲
- `examiner.md` 保持严格判分

### 研究生研讨式

适合：哲学、历史、政治、经济思想史。

特点：

- 先要求学习者重构文本或论证
- 再比较不同解释路径
- 保留开放问题，但要标清证据
- 适合原典和论文阅读

题库倾向：

- 论证重构
- 文本证据定位
- 解释分歧比较

### 严格考官式

适合：数学、考试、专业资格训练。

特点：

- 标准明确
- 答案必须可判定
- 错误类型细分
- 反馈短而具体

题库倾向：

- 定义题
- 证明题
- 计算题
- 反例题

### 案例教练式

适合：经济、交易、管理、法律、公共政策。

特点：

- 以真实或拟真场景为主
- 要求学习者先做判断，再解释规则
- 强调决策边界和风险
- 适合迁移训练

题库倾向：

- scenario
- boundary
- argument_analysis

### 复盘审计式

适合：交易、写作、项目管理、产品设计。

特点：

- 每次练习后必须复盘
- 区分结果错误和过程错误
- 记录可改进动作
- 强调下一次如何避免同类错误

teacher 调整：

- `homework_log.md` 增加“决策过程”和“修正动作”
- `examiner.md` 增加过程审计标准

### 历史叙事式

适合：历史、思想史、政治史。

特点：

- 先建立时间线
- 再解释结构因素和人物选择
- 允许多因素答案，但必须有证据层级
- 重视因果链和背景条件

题库倾向：

- 时间线重构
- 因果解释
- 史料辨析

### 写作工作坊式

适合：论文、非虚构、商业写作、创意写作。

特点：

- 每个技能点都要产出文本
- 反馈围绕读者、结构、论证和语言
- 修改比初稿更重要
- rubric 要能处理“可读但不够强”的答案

题库倾向：

- 改写题
- 诊断题
- 结构重排
- 风格转换

## 选择建议

如果不确定用哪一种，先按这个规则选：

- 学概念：苏格拉底式
- 学模型：案例教练式
- 学证明：严格考官式
- 学文本：研究生研讨式
- 学技能：复盘审计式
- 学写作：写作工作坊式

风格不是互斥的。一个系统可以在概念课使用苏格拉底式，在练习课使用严格考官式，在课后日志使用复盘审计式。
